# 二、总体架构

## 2.1 分层结构

```cpp
Platform Layer (SDL3)
        ↓
InputSystem（状态 + 事件）
        ↓
Action Mapping
        ↓
Input Context Stack
        ↓
Command System
        ↓
Editor Systems
        ↓
ECS（Transform / Camera）
        ↓
RenderGraph（含 Picking Pass）
```

---

## 2.2 核心设计思想

### ✅ 输入是“事件 + 状态”双系统

* Event：用于 UI / Console / Text
* State：用于游戏逻辑（持续输入）

---

### ✅ 输入必须解耦平台

* 不允许在引擎中直接使用 SDL_Event

---

### ✅ 输入驱动 Command，而不是直接改数据

```cpp
Input → Action → Command → ECS
```

---

### ✅ Editor 是 Input 的“上层系统”

# 三、Input System 设计

---

## 3.1 输入数据结构

### Key

```cpp
enum class Key
{
    W, A, S, D,
    Space,
    MouseLeft,
    MouseRight,
};
```

---

## 3.2 InputState（状态系统）

```cpp
struct InputState
{
    bool current[KeyCount];
    bool previous[KeyCount];

    float mouseX, mouseY;
    float mouseDX, mouseDY;
    float wheel;
};
```

---

## 3.3 帧更新机制

```cpp
BeginFrame():
    previous = current
    mouseDelta = 0
```

---

## 3.4 查询接口

```cpp

IsPressed()
IsReleased()
```

---

## 3.5 SDL → Input 转换


* 映射 Key

# 9.1 Ray Picking（基础）

流程：

```cpp
Screen → Ray → Scene → Intersect
```

---

## 缺点

* CPU 计算
* 不精确（复杂 mesh）

---

# 9.2 GPU Picking（推荐）

---

## 核心思想

> 渲染 ID Buffer，每个像素记录对象ID

---

## 数据流

```cpp
Render ID → Read Pixel → EntityID
```

---

## Vulkan 实现

### Image

```cpp
VK_FORMAT_R32_UINT
```

---

### Shader

```glsl
out uint entityID;
```

---

## 优化

### ✅ 只读取一个像素

### ✅ 多帧延迟读取

### ✅ staging buffer

---

## 问题

GPU → CPU 读取会阻塞
