# 引擎日志系统设计文档（Logging System Specification）

---

# 1. 概述

本日志系统基于 **spdlog** 构建，是引擎的核心调试基础设施（Debug Infrastructure），用于：

* 调试与错误定位
* 性能分析辅助
* 崩溃诊断
* GPU 调试联动
* 远程日志与控制

系统目标：

* 高性能（异步、低开销）
* 可扩展（多 sink）
* 可观测（与 profiler / GPU 工具联动）
* 可控制（运行时动态调整）

---

# 2. 总体架构

```text
Logging System
 ├── Logger Layer（Core / Renderer / App）
 ├── Sink Layer（Console / File / ImGui / Network）
 ├── Async System（线程池 + 队列）
 ├── Tag System（分类）
 ├── Compile-time Strip（编译期裁剪）
 ├── Crash System（崩溃处理 + 回溯）
 ├── Debug Integration（Vulkan / GPU Marker）
 └── Console System（运行时控制）
```

# 21. Threading Model（线程模型）

## 架构

```text
Main Thread
   ↓
Queue
   ↓
Worker Thread
   ↓
Sink
```

---

## 队列策略

| 策略             | 行为        |
| -------------- | --------- |
| block          | 阻塞线程      |
| overrun_oldest | 丢弃旧日志（推荐） |

---

## 要点

* 单 worker → 有序
* 多 worker → 可能乱序

---

## 生命周期

```cpp
spdlog::shutdown();
```

---

## 原则

### ✅

* IO 在后台线程
* 队列足够大

### ❌

* 主线程写文件
* 小队列

## 等级规范

| level | 用途    |
| ----- | ----- |

# 24. Logging Anti-Patterns（日志反模式）

---

## 24.1 过度日志

❌ 高频打印
👉 卡顿 + 噪声

---

## 24.2 日志信息不足

❌ 无上下文
👉 无法调试

---

## 24.3 无结构日志

❌ 不可搜索

---

## 24.4 无关联日志

❌ 无 trace id

---

## 24.5 Log-and-Throw

❌ 重复日志

---

## 24.6 无意义日志

❌ "Unknown error"

---

## 24.7 热路径日志

❌ 每帧打印

---

## 24.8 同步日志

❌ 阻塞线程

---

## 24.9 错误等级

❌ warn 写成 error

---

## 24.10 日志替代 profiling

❌ 不准确

---

## 24.11 日志耦合逻辑

❌ 改变程序行为

---

## 24.12 日志分散

❌ printf + spdlog 混用

---

## 24.13 日志无限增长

❌ 不做轮转

---

## 24.14 总结

反模式会导致：

```text
性能下降
可读性差
调试困难
技术债累积
```

---

## 24.15 原则

### ✅ DO

* 有价值日志
* 正确等级
* 完整上下文

### ❌ DON'T

* 热路径 logging
* 滥用 trace
* 无意义日志
