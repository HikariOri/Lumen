# 🧠 一句话目标

👉 **Material = 一组 GPU 参数 + 一组纹理 + 一个 descriptor set（或索引）**

---

# 🚀 一、最小可用设计（你现在就该这样写）

---

## ✔ CPU 材质结构（强烈推荐）

```cpp
struct PBRMaterial {
    // ===== 常量参数（UBO / push constant）=====
    glm::vec4 baseColorFactor {1.0f}; // RGBA
    float metallicFactor  {1.0f};
    float roughnessFactor {1.0f};

    glm::vec3 emissiveFactor {0.0f};

    // ===== 纹理（句柄 / 指针 / index）=====
    Texture* baseColorTex            = nullptr;
    Texture* metallicRoughnessTex    = nullptr;
    Texture* normalTex               = nullptr;
    Texture* occlusionTex            = nullptr;
    Texture* emissiveTex             = nullptr;

    // ===== 状态 =====
    bool doubleSided = false;
    int alphaMode = 0; // OPAQUE / MASK / BLEND
};
```

---

# 🧠 二、shader 对应（标准 glTF 写法）

```glsl
vec4 baseColor = texture(baseColorTex, uv) * baseColorFactor;

float metallic  = texture(mrTex, uv).b * metallicFactor;
float roughness = texture(mrTex, uv).g * roughnessFactor;

vec3 normal = getNormalFromMap(...);
vec3 emissive = texture(emissiveTex, uv).rgb * emissiveFactor;
```

👉 你可以看到：

```text
texture × factor
```

这是整个系统的核心

---

# 🚀 三、Descriptor Set 设计（非常关键）

你有两个选择👇

---

## 🟢 方案 A：每个 Material 一个 Descriptor Set（推荐起步）

```text
set = 1 (material)

binding 0 → baseColorTex
binding 1 → metallicRoughnessTex
binding 2 → normalTex
binding 3 → occlusionTex
binding 4 → emissiveTex
```

---

### ✔ 优点

* 简单
* 易 debug
* 和 glTF 一一对应

---

### ❌ 缺点

* descriptor 数量多
* bind 成本高

---

## 🔵 方案 B：Bindless / Texture Array（进阶）

```text
所有纹理 → 一个大数组
material → 存 index
```

---

```cpp
struct MaterialGPU {
    int baseColorTexIndex;
    int mrTexIndex;
    int normalTexIndex;
};
```

---

```glsl
texture(sampler2DArray, vec3(uv, index))
```

👉 这是现代引擎方案（你后面一定会走这条路）

---

# 🚀 四、Material → GPU 数据分离（重要设计）

---

## ❗ 不要直接把 CPU struct 丢进 GPU

👉 正确做法：

---

## ✔ GPU 结构（紧凑版）

```cpp
struct MaterialGPU {
    glm::vec4 baseColorFactor;

    float metallicFactor;
    float roughnessFactor;

    int baseColorTex;
    int mrTex;
    int normalTex;
};
```

---

👉 放进：

```text
SSBO / UBO / push constant
```

---

# 🚀 五、Mesh / Primitive 如何关联 Material

你前面问的点，这里串起来👇

---

## ✔ glTF 结构

```text
Mesh
  └── Primitive
         └── Material
```

---

## ✔ 你的设计

```cpp
struct Primitive {
    Buffer vertexBuffer;
    Buffer indexBuffer;

    Material* material;
};
```

---

👉 渲染时：

```cpp
bind(material)
draw(primitive)
```

---

# 🚀 六、渲染流程（完整）

```text
for each mesh:
  for each primitive:
    bind pipeline
    bind material descriptor
    push material constants
    draw
```

---

# 🧠 七、关键优化点（你很快会遇到）

---

## ❗ 1. Material 排序（减少 bind）

```text
按 material 排序 draw call
```

---

## ❗ 2. Descriptor 缓存

```text
相同 material → 共享 descriptor set
```

---

## ❗ 3. 纹理默认值（非常重要）

当 texture 不存在：

| 类型                | 默认          |
| ----------------- | ----------- |
| baseColor         | 白           |
| normal            | (0.5,0.5,1) |
| metallicRoughness | (1,1)       |
| AO                | 1           |
| emissive          | 黑           |

---

👉 否则 shader 会炸

---

# 🧠 一句话总结

👉 **材质类的本质是：把 glTF 的“texture × factor”映射到 GPU（descriptor + 常量），并让每个 primitive 能引用它。**
